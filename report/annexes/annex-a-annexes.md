# Annex A: Annexes Index

Los anexos del reporte Nexa AV2 se separan en archivos Markdown independientes para facilitar el mantenimiento Docs-as-Code, reducir el tamaño de cada sección y conservar trazabilidad entre evidencias, repositorios, despliegues, validación y.

Esta reorganización no agrega evidencias nuevas ni cambia el sentido de las evidencias existentes. Las rutas de imágenes se mantienen relativas a `report/annexes/`, por lo que las referencias `../assets/...` continúan funcionando dentro de los anexos separados.

| Anexo | Archivo | Contenido |
|---|---|---|
| Annex A: Annexes Index | `report/annexes/annex-a-annexes.md` | Índice general de anexos y criterio de organización. |
| Annex B: Video Evidence | `report/annexes/annex-b-video-evidence.md` | Evidencias audiovisuales, enlaces Stream/SharePoint existentes y pendientes controlados de videos AV2. |
| Annex C: Validation Evidence | `report/annexes/annex-c-validation-evidence.md` | Evidencia de Needfinding, Validation Interviews AV2 y evaluaciones heurísticas preparadas para validación posterior. |
| Annex D: GitHub Repository Evidence | `report/annexes/annex-d-github-repository-evidence.md` | Repositorios, releases, commits, branches, GitHub Insights y evidencias de colaboración documental/técnica. |
| Annex E: Deployment and Services Evidence | `report/annexes/annex-e-deployment-and-services-evidence.md` | Jira, Render, GitHub Pages, Swagger/OpenAPI, ejecución WebApp, estructura backend y servicios. |
| Annex F: AV2 Pending and Controlled Evidence | `report/annexes/annex-f-av2-pending-controlled-evidence.md` | Pendientes reales de AV2 que requieren evidencia posterior. |

> *Nota:* Los pendientes reales se mantienen centralizados en Annex F para evitar declarar evidencias no ejecutadas o no publicadas.
