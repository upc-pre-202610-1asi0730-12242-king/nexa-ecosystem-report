# Annex A: Annexes Index

Los anexos del reporte Nexa AV2 se separan en archivos Markdown independientes para facilitar el mantenimiento Docs-as-Code, reducir el tamaño de cada sección y conservar trazabilidad entre evidencias, repositorios, despliegues, validación y material audiovisual.

Esta reorganización ordena las evidencias disponibles del corte AV2 sin declarar artefactos no verificados. Las rutas de imágenes se mantienen relativas a `report/annexes/`, por lo que las referencias `../assets/...` continúan funcionando dentro de los anexos separados.

| Anexo | Archivo | Contenido |
|---|---|---|
| Annex A: Annexes Index | `report/annexes/annex-a-annexes.md` | Índice general de anexos y criterio de organización. |
| Annex B: Video Evidence | `report/annexes/annex-b-video-evidence.md` | Evidencias audiovisuales, enlaces Stream/SharePoint y capturas disponibles de videos AV2. |
| Annex C: Validation Evidence | `report/annexes/annex-c-validation-evidence.md` | Evidencia de Needfinding, Validation Interviews AV2 y evaluaciones heurísticas registradas. |
| Annex D: GitHub Repository Evidence | `report/annexes/annex-d-github-repository-evidence.md` | Repositorios, releases, commits, branches, GitHub Insights y evidencias de colaboración documental/técnica. |
| Annex E: Deployment and Services Evidence | `report/annexes/annex-e-deployment-and-services-evidence.md` | Jira, Render, GitHub Pages, Swagger/OpenAPI, ejecución WebApp, estructura backend y servicios. |
| Annex F: AV2 Controlled Evidence | `report/annexes/annex-f-av2-controlled-evidence.md` | Evidencias AV2 disponibles y límites declarados del corte documental. |

> *Nota:* Los límites del corte AV2 se mantienen centralizados en Annex F para evitar declarar resultados por encima de la evidencia disponible.
