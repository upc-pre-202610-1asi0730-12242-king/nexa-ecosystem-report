# Repository Structure

Below is the repository mapping for the Nexa ecosystem, listing target roles and engineering links:

| Repository | Purpose | Primary Tech Stack | Live Site / Docs |
| :--- | :--- | :---: | :--- |
| **Nexa Report** | Academic report and requirements research. | Markdown (Docs-as-Code) | [Report Link](https://github.com/upc-pre-202610-1asi0730-12242-king/nexa-ecosystem-report) |
| **Nexa Website** | Landing website and marketing gateway. | HTML5, CSS3, Vanilla JS | [Live Website](https://upc-pre-202610-1asi0730-12242-king.github.io/nexa-website/) |
| **Nexa WebApp** | Operation dashboard frontend interface. | Vue 3, Vite, PrimeVue, Pinia | [Live WebApp](https://nexa-webapp.onrender.com) |
| **Nexa Platform** | Backend API and database service layer. | C# / ASP.NET Core, EF Core | [API Swagger](https://nexa-platform-api.onrender.com) |

## Report Folder Structure
- `/report/front-matter`: Version history logs and Outcomes index files.
- `/report/10-chapter-1-introduction` to `50-chapter-5-implementation-validation-deployment`: Main document body.
- `/releases`: Release notes directory.
- `/wiki`: Local copy of this wiki.
