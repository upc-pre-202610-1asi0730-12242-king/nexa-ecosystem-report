# Quality and Security

Nexa documentation and software artifacts follow strict quality assurance and secure coding guidelines.

## Security Policies
- **No Credentials Commits**: Never commit API keys, connection strings, or system passwords. Use environment variables (`.env`).
- **Standardized Accounts**: Test login accounts must be cleared of private credentials (defaulting to academic logins like `buyer@nexa.com`, `Nexa2026!`).
- **Vulnerability Reporting**: Report any vulnerability immediately to [diego64g284@gmail.com](mailto:diego64g284@gmail.com) following `.github/SECURITY.md`.

## Document Quality Standards
- Check spelling and grammar before committing.
- Ensure all tables render cleanly on standard Markdown viewers.
- Verify image references before tagging a new release.
